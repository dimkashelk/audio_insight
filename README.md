# AudioInsight AI
> Микросервисная система автоматической транскрибации и интеллектуальной суммаризации учебных материалов

## О проекте
Автономное локальное решение для преобразования аудио- и видеозаписей лекций, вебинаров и встреч в структурированные текстовые отчёты. Система объединяет локальное распознавание речи (OpenAI Whisper) и генеративную нейросеть (Gemma 3, дообученная через LoRA) для создания кратких резюме и выделения ключевых терминов **без отправки данных на внешние серверы**.

Проект разработан в рамках курсовой работы по дисциплине `Глубокое обучение`.

---

## Возможности
- **Локальная транскрибация**: Поддержка `.mp3`, `.wav`, `.mp4`, `.m4a`, `.webm` через OpenAI Whisper (`turbo`/`base`)
- **Интеллектуальная суммаризация**: Генерация резюме (до 5 пунктов) + ключевые термины через дообученную LLM
- **Асинхронный пайплайн**: Очередь задач на Celery + Redis, неблокирующий UI с детализированным прогрессом
- **Полная контейнеризация**: `docker-compose` для воспроизводимого развёртывания
- **Конфиденциальность**: Все вычисления выполняются локально, данные не покидают устройство

## 🚀 Быстрый старт

### 1. Предварительные требования
- Установленный [Docker Desktop](https://www.docker.com/products/docker-desktop/)
- Установленный [Ollama](https://ollama.com/) (локально на хосте)
- macOS / Linux / Windows (WSL2), 16+ ГБ ОЗУ

### 2. Подготовка модели Ollama
Система использует кастомную модель `gemma-summarizer` с жёстким системным промптом. Создайте её один раз:
```bash
# Скачайте базовую модель
ollama pull gemma2:2b

# Создайте кастомную модель (файл Modelfile лежит в корне репозитория)
ollama create gemma-summarizer -f Modelfile
```

### 3. Запуск стека
```bash
# 1. Клонируйте репозиторий
git clone <URL_РЕПОЗИТОРИЯ>
cd audio_insight

# 2. Запустите все сервисы
docker compose up -d --build

# 3. Откройте интерфейс
# Frontend: http://localhost:8501
# Backend Docs: http://localhost:8000/docs
# ML Service: http://localhost:8001/health
```

## Дообучение модели
Для достижения стабильного формата вывода (1. ...\n2. ...\n\nКлючевые термины: ...) был проведён цикл экспериментов:
1. Генерация датасета: ~150 пар «транскрипт → эталонное резюме» с автоматической разметкой и ручной валидацией.
2. Сравнительные эксперименты:
* Qwen 2.5-1.5B: Быстрый сход loss, но сильное переобучение, зацикливание фраз и нарушение формата на длинных входах. 
* Gemma 2-2B: Стабильное следование структуре, но высокая перплексия на тесте (13.11) из-за неоптимального LR.  
6. Метод: LoRA (rank=8, alpha=16) через mlx-lm на Apple Silicon (24 ГБ RAM).
